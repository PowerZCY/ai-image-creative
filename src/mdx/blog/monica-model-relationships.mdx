## 1. 目的

本文用于在落地 Prisma schema 前，统一说明当前已有基础数据表和 Monica AI 新增业务表之间的数据流向。

重点回答：

- 每张表对应哪个中文业务概念。
- 用户、积分、主题、生成、投稿、公开展示之间如何流转。
- 哪些表是现有基础能力，哪些表是 Monica AI 新增业务能力。
- 哪些关系决定 API 鉴权、归属校验和索引设计。

## 2. 表名与中文业务名称

### 2.1 已有基础表

<CheetTable title="已有基础表">

| 表名 / Model | 中文业务名称 | 当前职责 |
| --- | --- | --- |
| `User` | 用户 | 用户身份主表，承接匿名用户、注册用户、Clerk 用户、Stripe Customer |
| `Credit` | 积分余额 | 用户当前可用积分余额，区分免费、订阅、一次性购买积分 |
| `CreditAuditLog` | 积分流水 | 记录积分发放、消耗、调整、清空等审计流水 |
| `Subscription` | 订阅 | 用户订阅状态和订阅周期 |
| `Transaction` | 订单交易 | Stripe 等支付订单、付款状态、授予积分 |
| `UserBackup` | 用户备份 | 用户删除或迁移时的备份记录 |
| `Apilog` | API 日志 | 外部系统或关键 API 调用日志 |

</CheetTable>

### 2.2 Monica AI 新增业务表

<CheetTable title="Monica AI 新增业务表">

| 表名 / Model | 中文业务名称 | 当前职责 |
| --- | --- | --- |
| `ThemeSubmission` | 主题提交 | 用户或管理员提交每日主题想法，进入审核和候选池 |
| `Theme` | 正式主题 | 已采纳、排期或发布的每日主题 |
| `ReferenceImage` | 参考图 | 用户上传的本地参考图片，作为生成输入素材 |
| `GenerationJob` | 生成任务 | 图片生成、变体、编辑的异步任务记录 |
| `GeneratedImage` | 生成图片 | AI 生成出的用户私有图片资产 |
| `ImageSubmission` | 作品投稿 | 用户将生成图片提交到主题的审核记录 |
| `ReviewRecord` | 审核记录 | 自动审核和人工审核历史 |
| `PublicImage` | 公开图片 | 审核通过后进入 Explore / Theme Gallery 的公开展示记录 |
| `ImageLike` | 图片点赞 | 用户对公开图片的点赞关系 |
| `ImageSave` | 图片收藏 | 用户对公开图片的收藏关系 |

</CheetTable>

## 4. 主业务链路

### 4.1 主题生产链路

<Mermaid
  title="4.1 主题生产链路"
  chart={`
flowchart LR
  User[User-用户] --> ThemeSubmission[ThemeSubmission-主题提交]
  ThemeSubmission --> Review[审核/编辑]
  Review --> Theme[Theme-正式主题]
  ThemeSubmission --> CreditAuditLog[CreditAuditLog-积分流水]
`}
/>

说明：

- 普通用户和管理员都通过 `ThemeSubmission-主题提交` 进入主题生产流程。
- 管理员提交时也是一条链路，只是审核路径更快。
- `Theme-正式主题` 只表示已采纳、排期或发布的主题。
- 用户提交主题获得奖励时，不新建奖励表，写入 `CreditAuditLog-积分流水`。

### 4.2 图片生成链路

<Mermaid
  title="4.2 图片生成链路"
  chart={`
flowchart LR
  User[User-用户] --> GenerationJob[GenerationJob-生成任务]
  Theme[Theme-正式主题] --> GenerationJob
  User --> ReferenceImage[ReferenceImage-参考图]
  ReferenceImage --> GenerationJob[GenerationJob-生成任务]
  GenerationJob --> CreditAuditLog[CreditAuditLog-积分流水]
  GenerationJob --> GeneratedImage[GeneratedImage-生成图片]
`}
/>

说明：

- Assistant 中间过程不持久化；最终用于生成的 prompt 记录在 `GenerationJob-生成任务`。
- `ReferenceImage-参考图` 是用户上传的输入素材，不是作品。
- `GenerationJob-生成任务` 是调用 AI、记录供应商状态和预计/实际积分消耗的任务表。
- `GeneratedImage-生成图片` 是 Studio 中展示的用户私有图片资产。
- 生成任务创建时预扣减积分；失败、拦截或取消时由积分服务按 `GenerationJob.jobId` 返还。扣费和返还通过 `CreditAuditLog-积分流水` 追溯，余额仍由 `Credit-积分余额` 管理。

### 4.3 投稿与公开展示链路

<Mermaid
  title="4.3 投稿与公开展示链路"
  chart={`
flowchart LR
  User[User-用户] --> GeneratedImage[GeneratedImage-生成图片]
  GeneratedImage --> ImageSubmission[ImageSubmission-作品投稿]
  Theme[Theme-正式主题] --> ImageSubmission
  ImageSubmission --> ReviewRecord[ReviewRecord-审核记录]
  ImageSubmission --> PublicImage[PublicImage-公开图片]
  GeneratedImage --> PublicImage
  PublicImage --> ImageLike[ImageLike-图片点赞]
  PublicImage --> ImageSave[ImageSave-图片收藏]
`}
/>

说明：

- 生成图片不会自动公开。
- 用户主动提交后创建 `ImageSubmission-作品投稿`。
- 投稿审核过程写入 `ReviewRecord-审核记录`。
- 通过审核后创建 `PublicImage-公开图片`。
- `PublicImage-公开图片` 不重复存图片文件，它引用 `GeneratedImage-生成图片`，只承载公开展示、排序、点赞、收藏相关数据。

### 4.4 支付与积分链路

<Mermaid
  title="4.4 支付与积分链路"
  chart={`
flowchart LR
  User[User-用户] --> Transaction[Transaction-订单交易]
  Transaction --> Subscription[Subscription-订阅]
  Transaction --> Credit[Credit-积分余额]
  Transaction --> CreditAuditLog[CreditAuditLog-积分流水]
  GenerationJob[GenerationJob-生成任务] --> CreditAuditLog
  ThemeSubmission[ThemeSubmission-主题提交] --> CreditAuditLog
`}
/>

说明：

- 购买、订阅、支付状态继续使用现有 `Transaction-订单交易` 和 `Subscription-订阅`。
- 用户当前余额继续使用 `Credit-积分余额`。
- 所有积分变动继续使用 `CreditAuditLog-积分流水`。
- 图片生成预扣减、最终消耗或失败返还时，`CreditAuditLog.operationReferId` 可记录 `GenerationJob.jobId`。
- 主题提交奖励积分时，`CreditAuditLog.operationReferId` 可记录 `ThemeSubmission.themeSubmissionId`。

## 7. 关键设计结论

- `User-用户` 是所有业务数据归属的根。
- `Credit-积分余额`、`CreditAuditLog-积分流水`、`Transaction-订单交易`、`Subscription-订阅` 已经完备，不需要为 Monica AI 新建积分或订单表。
- `ThemeSubmission-主题提交` 是主题来源，普通用户和管理员走同一条主题生产链路。
- `Theme-正式主题` 是创作上下文和投稿目标。
- `GenerationJob-生成任务` 是 AI 供应商调用、异步状态、预计消耗和实际消耗记录的核心任务表。
- `GeneratedImage-生成图片` 是 My Studio 的核心资产。
- `ImageSubmission-作品投稿` 是从私有图进入公开展示前的审核层。
- `PublicImage-公开图片` 是公开展示投影，不重复存储图片资产。
- `ReferenceImage-参考图` 是生成输入素材，不是作品。
- `ImageLike-图片点赞` 和 `ImageSave-图片收藏` 是用户与公开图片之间的关系表。
